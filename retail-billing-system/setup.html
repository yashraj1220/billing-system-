2 Online Mode with MySQL (For Multi-Device Access)

Setup MySQL database for cloud sync and multi-device access.

Prerequisites:

  • Web server (Apache/Nginx)
  • PHP 7.0+ with MySQL extension
  • MySQL 5.7+ database

Setup Steps:

Step A: Create Database

mysql -u root -p < database.sql

Or import via phpMyAdmin:

  1. Login to phpMyAdmin
  2. Create new database: retail_billing
  3. Import database.sql file

Step B: Configure API

Edit api.php and update database credentials:

define('DB_HOST', 'localhost'); define('DB_USER', 'your_username'); define('DB_PASS', 'your_password'); define('DB_NAME', 'retail_billing');

Step C: Upload Files

Upload all files to your web server's public directory (htdocs/public_html)

Step D: Test API

Visit: http://yourdomain.com/api.php?action=export

Should return JSON response with success status

Step E: Enable Sync in Application

Open index.html and add sync.js before closing body tag:

<script src="sync.js"></script>

üîÑ Backup & Restore

Manual Backup (Offline Mode)

  1. Open browser console (F12)
  2. Run: exportAllData().then(data => console.log(JSON.stringify(data)))
  3. Copy the JSON output
  4. Save to text file

Restore from Backup

  1. Open browser console (F12)
  2. Paste your backup JSON into variable
  3. Run: importAllData(yourBackupData)

Automatic Backup (MySQL Mode)

Data is automatically synced to MySQL database. Schedule regular MySQL backups using:

mysqldump -u username -p retail_billing > backup_$(date +%Y%m%d).sql

üõ†Ô∏è Troubleshooting

Common Issues

Issue: Data not saving

  • Check if IndexedDB is enabled in browser
  • Clear browser cache and retry
  • Check browser console for errors (F12)

Issue: WhatsApp not opening

  • Verify phone number format (+countrycode)
  • Enable pop-ups for the site
  • Ensure WhatsApp is installed

Issue: MySQL sync not working

  • Check database credentials in api.php
  • Verify MySQL service is running
  • Check PHP error logs
  • Ensure CORS headers are set correctly

Issue: Print not working

  • Allow pop-ups for the site
  • Try different browser (Chrome recommended)
  • Check if JavaScript is enabled
